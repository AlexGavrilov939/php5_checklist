Basic syntax (answers)

PHP tags

  1. <?php ... ?>
    <? ... ?> (php.ini short_open_tag must be enabled or if PHP was configured with the --enable-short-tags option)
    <?= ... ?> (short_open_tag ini setting or always available from 5.4.0 version)
    <% ... %> ASP styled tags (are removed from 7.0.0)
    <%= ... %> ASP styled tags (are removed from 7.0.0)
    <script language="php"> (are removed from 7.0.0)
  2. Yes. This prevents accidental whitespace or new lines being added after the PHP closing tag, which may cause unwanted effects because PHP will start output buffering when there is no intention from the programmer to send any output at that point in the script.

Notes:

  • <?php/*blah*/ echo "a"?> would not work (white needed after "php")
  • The last tag "?>" deletes the end of the line.
